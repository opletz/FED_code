WAI(Web Accessibility Initiative) was created in 1997 by W3C(The World Wide Web Consortium) to address improving accessibility to web content. Web Accessibility attempts to present web content with consideration to individuals with disabilities or physical, visual or audible impairments.

Creating a more accessible website consist of writing content for alternate forms of interpretation. From a visual perspective this would consist of writing text descriptions for visual elements such as images,graphs and charts. It means designing a webpage using stronger contrast, adjustable text size and making designs decisions that may help present contents to individuals with color blindness.

Web Accessibility covers an extremely large amount of information but a few things that can help make a website more accessible include the following.

  • Create alt attributes on all HTML img tags to describe the visual elements.
  • Design with consideration to text size, color contrast and color blindness.
  • Support intuitive navigation with support for both the mouse and keyboard.
  • Assign roles to HTML elements to help screen readers separate content,navigation,etc.
  • Refrain from using links to "here","this" and similar. Links should be descriptive.

To see examples of the alt attribute and Document Landmark Roles look within the source code of this document look for HTML img tags similar to <img alt="some text description"> and roles such as <nav role="navigation">. Although HTML5 tags such as <nav> may be automatically identified by newer browsers it doesn't hurt to add roles for screen readers or browsers that don't fully understand the new HTML5 semantics.
